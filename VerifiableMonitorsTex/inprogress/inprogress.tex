\begin{abstract}
    Runtime monitors for hidden Markov models obtain a sequence of observations and must classify whether the system is currently deemed safe.
    A system is deemed safe, if the probability of being in a dangerous state conditioned on these observations is below some threshold.
    The key contribution is the first correct-by-construction synthesis method for such monitors, represented as finite state machines. The contribution combines four ingredients:
    First, we establish the coNP-hardness of checking whether a finite state machine is a correct monitor, i.e., a monitor without misclassifications.
    Second, we provide a reduction that reformulates the search for misclassifications into a standard probabilistic system synthesis problem.
    Third, we integrate the verification routine into an active automata learning routine to synthesise correct monitors.
    Fourth, we provide a prototypical implementation that shows the feasibility and limitations of the approach on a series of benchmarks.

\section{Introduction}

Runtime assurance is an essential ingredient in the deployment of safe autonomous systems.
Runtime monitors provide assurance by flagging potentially dangerous system behavior by observing a system execution.
More precisely, a monitor receives a stream of observations about the system and outputs a verdict, e.g., it raises an alarm that the system has left some safety envelope. A monitor is correct if it correctly raises such alarms based on a formal specification.
Various challenges in creating correct runtime monitors for (semi-)autonomous systems have been identified~\cite{}, such as:
(1)~the state of the system is only partially observable, i.e., the stream of observations comes from sensor readings and does not uniquely identify the state of a system,
(2)~the behavior of the system and/or the sensors may be subject to probabilistic uncertainty,
(3)~the monitor itself is subject to resource constraints (time, memory, etc), and
(4)~the monitor is itself safety-critical and should therefore be subject to extensive validation.
Challenges (1,2) can be (partially) addressed  by modelling the system as a hidden Markov model (HMM), Challenges (3,4) can be (partially)\li{What does partially mean here? Perhaps a footnote that says you discuss this somewhere?} addressed by representing a monitor as, e.g., a (small) finite state machine.
Concretely, this paper focuses on the following question:
\emph{Is a given finite state machine a correct monitor for a given HMM?}
The paper studies the complexity of this problem, provides a practical verification procedure, and embeds this verification procedure into a learning framework.

\paragraph{What are HMMs?}
In this paper, we assume that the system including its sensors is adequately described as a discrete Hidden Markov Model (HMM).
Markov chains (MCs) describe system behavior subject to probabilistic uncertainty.
Paths through an MC are sequences of states that describe system executions.
% MCs can be seen as specifying a distribution over these paths. 
HMMs extend MCs by labelling their states with colors (aka \emph{observations}).
Intuitively, the observations can be used to model the information that the monitor receives.
In HMMs, every path can be lifted to a sequence of observations, which we call a \emph{trace}.
The trace associated to a system execution describes the information received by the monitor.


% where every state is labelled with a distribution over observations, see Fig.~\ref{}. Paths are sequences of states and MCs can be seen as specifying a distribution over these paths. An HMM extends MCs by assigning to every state a colour (an \emph{observation}). A trace is a sequence of these observations. 

\paragraph{Risk of a trace.}\li{Maybe move after What are correct monitors. I feel like "what are correct monitors" is a question that immediately comes up, while risk of a trace is not.}
In runtime monitoring, an essential task\li{How does this relate to the problem statement?} is to estimate whether the current system is dangerous. In the simplest setting, a specification describes whether a set of states is dangerous. The risk of a trace is then the probability of ending in a dangerous state, conditioned on the fact that the system execution matches the trace.
For a given trace, we may compute this risk, e.g., either via a reduction to model checking~\cite{} or by a (forward) filtering that tracks a distribution over the current states~\cite{}.


\paragraph{What are correct monitors?}
Often, we are not interested in this precise risk, but rather want to  partition the set of horizon-bounded traces into safe and unsafe traces:
Unsafe traces are traces whose risk does exceed some fixed \emph{acceptable threshold}, all other traces are safe.
Indeed, a \emph{correct} monitor obtains a trace and raises an alarm iff a trace is not safe. The key verification problem in this paper asks whether a monitor, represented as a deterministic finite automaton, accepts (i.e., raises an alarm) exactly the unsafe traces. We note that in this paper, we only consider traces that are bounded by some fixed horizon.

\paragraph{Novel complexity results.}
Deciding whether a monitor is correct is coNP-complete (\Cref{lem:complexity}).
In particular, we study the dual problem that asks whether a monitor misclassifies at least one trace.
We consider a simpler variant of this problem that asks whether there is a bounded trace in the HMM that is unsafe. \footnote{The results also apply to checking for safe traces.}
The problem is clearly in NP (assuming that the bound is unary encoded).
We show that this simplified problem is also strongly NP-hard (\Cref{lem:ctpdecnphard}) and the corresponding optimization problem, that asks to approximate the risk of the trace with the heighest risk, is APX-hard (\Cref{lem:ctpapxhard}).
Intuitively, APX-hardness means that the existence of an efficient approximation algorithm would have major complexity-theoretic implications.


\paragraph{Our verification approach.}
While deciding whether a monitor is correct is thus in general not tractable, we suggest to utilise recent advances in the synthesis of probabilistic systems.
We concentrate on proving the absence of missed alarms, i.e., we concentrate on showing that the monitor correctly identifies every unsafe trace. A similar reduction works for showing the absence of spurious\li{The term spurious alarm only appreas here, maybe describe it instead?} alarm.
First, for a single trace, computing the risk can be reduced to computing a reachability probability in a MC that is intuitively a kind of product between a DFA that accepts exactly the trace and the HMM~\cite{}.
Thus, it is natural to reduce our problem to the question: Is there a DFA (accepting exactly one trace, accepted by our monitor)
such that the probability in the product MC exceeds a threshold\li{emph?}.
If the answer is no, then the monitor has no-missed-alarms.
This decision problem can be solved using (exact) probabilistic model synthesis methods~\cite{} as implemented in the tool Paynt~\cite{}\footnote{Alternatively, in \cref{sec:monver}, we show that we can construct a product of an acyclic DFA accepting sets of traces and the HMM. However, this yields partially observable MDP, where we must search for memoryless policies, similar to a construction in~\cite{DBLP:conf/tacas/BadingsVJSJ24}. This problem is equivalent to the synthesis problem and can therefore also be solved using Paynt~\cite{DBLP:conf/uai/Andriushchenko022}.}. We discuss our verification approach in \Cref{sec:monver}.

\paragraph{Our learning approach. }
Ultimately, we do not only want to verify the correctness of a monitor, but we want to construct such a monitor.
Above, we already mentioned that the monitors are formal languages\li{This was not mentioned yet}. When considering bounded trace, these monitors are regular, and can thus be captured using DFAs.
Inspired by a recent short paper~\cite{}, we investigate learning these monitors:
We choose to do this using active automata learning~\cite{DBLP:journals/iandc/Angluin87,DBLP:journals/cacm/Vaandrager17}, which closely relates to other popular oracle-guided inductive synthesis loops~\cite{}.
To use active automata learning (AAL),
we must provide a \emph{membership oracle} that decides for a single trace whether the risk of the trace exceeds a threshold and an \emph{equivalence oracle} that decides whether a given hypothesis monitor is indeed correct. The membership oracle can be implemented by computing the risk for a single trace as above and comparing it to a threshold, while the equivalence oracle can be implemented by checking for the absence of missed and false alarms.
Practically, some minor modifications are necessary to embed our approach in  AAL framework: We have to handle the finite horizon, resolve some traces that are not relevant for the monitor, and we combine conformance and equivalence queries to boost performance.

\paragraph{Contributions.}
In summary, this paper provides a first framework to learn runtime monitors --- encoded as finite automata --- that are verified to be correct with respect to a given HMM. The main contributions are as follows:
(1)~We study monitor verification and show complexity-theoretic hardness of the underlying problem, even in an approximative setting.
(2)~We reduce the monitor verification to probabilistic model synthesis problems that exhaustively search for a counterexample.
(3)~We learn monitors by using the verifier above to answer equivalence queries in conjunction with active automata learning.
(4)~We demonstrate the feasibility and limitations of both the verifier and the learner on a series of benchmarks. Our prototype can handle HMMs with thousands of states, up to a hundred observations, and monitors with over 100 states.

\paragraph{DFAs.}
A \emph{deterministic finite automaton} (DFA) is a tuple $\dfa=\left(\StsD, \alphabet, \trans, \init, \final\right)$. $\StsD$ is a finite set of \emph{states}, $\alphabet$ is an \emph{alphabet}, $\trans\colon \StsD \times \alphabet \rightharpoonup \StsD$ is a (partial) \emph{transition function}, $\init$ is the \emph{initial state}, and $\final \subseteq \StsD$ is a set of \emph{accept} (or: \emph{final})\sj{Shall we just only use final?} states. %DFAs recognize regular languages:
Let $\varepsilon$ denote the empty word. We lift the transition relation to words, which is defined recursively: $\trans^*(q, \varepsilon) = \varepsilon$ if $a \in \alphabet$ and $\trans^*(q, w \cdot a) = \trans(\trans^*(q,w), a)$. The \emph{language} $\lang{\dfa}$ of a DFA $\dfa$ is the set of all words that end in a final state of the $\dfa$, $\lang{\dfa} = \{ w \in \alphabet^* \mid \trans^*(\init, w) \in \final \}$. We say that $\dfa$ accepts $w \in \lang{\dfa}$.


\subsection{Models}
We introduce MDPs and HMMs: The former are integral part of our approach, and HMMs are crucial to the problem statement. Details can be found in \cite{baierPrinciplesModelChecking2008}.


\begin{definition}[MDP]
    A \emph{Markov decision process} (MDP) is a tuple $\left(\Sts, \init, \Act, \ptrans \right)$ with a  countable nonempty set  $\Sts$ of \emph{states},  the \emph{initial state} $\init \in \Sts$, the \emph{partial transition function} $\ptrans\colon \Sts \times \Act \rightharpoonup \Distr(\Sts)$.
\end{definition}
We use $\Act(s) = \{ \act \mid \ptrans(s, \act) \neq \bot \}$ as the set of \emph{enabled actions}. We assume no deadlocks, i.e., for every $s \in \Sts$, $\Act(s) \neq \emptyset$.
A path $\pi$ is a sequence $s_0\act_0s_1\hdots \in (\Sts \times \Act)^{*} \times \Sts$, such that $a_i \in \Act(s_i)$ and $P(s_i, a_i)(s_{i+1}) > 0$ for every $i \geq 0$. %The set of all infinite paths is $\Pi^\mc$. 
The set of paths in MDP $\mdp$ is denoted as $\Pi^\mdp$, the set of finite paths are $\Pi_{fin}^\mdp$, the set of paths of at most length $h$ are $\Pi_h^\mdp$, and the set of paths of exactly length $h$ are $\Pi_{=h}^\mdp$. We drop $\mdp$ whenever it is clear from the context.\sj{Do we?}\sj{Check consistency of subscript/superscript!!}

A \emph{Markov chain} (MC) is an MDP where $|\Act(s)| = 1$ for every state $s \in \Sts$. We simplify notation and write MCs as tuple $\left(\Sts, \init,  \ptrans \right)$ and write $\ptrans(s)$ to refer to the unique distribution $\ptrans(s, \act)$.  Paths in a MC can be simplified to be sequences of (only) states.
The probability measure $\Prob^{\mc}$ of an MC $\mc$ is the unique probability measure following from the canonical $\sigma$-algebra associate with $\mc$.

\begin{definition}[Missed/False alarms]

\end{definition}

\begin{definition}[Correct monitor]\label{def:cormon}
    Given thresholds $\lambda_s,\lambda_u$ and horizon $h$, a monitor $\dfa$ for $\mc$ is \emph{correct} if $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa) = \emptyset = \falsealarms{\mc,\lambda_u}{\leq h}\!(\dfa)$.
\end{definition}
A correct monitor raises an alarm for all unsafe traces and for no safe trace, i.e., missed alarms are false negatives, while false alarms are false positives.
\begin{corollary}%[Correct monitor]
    A monitor $\dfa$ is correct iff $ \unsafetraces{\mc,\lambda_u}{\leq h} \subseteq \lang{\dfa} \subseteq \alphabet^* \setminus \safetraces{\mc, \lambda_s}{\leq h}$.% is a correct monitor.
\end{corollary}

\section{Monitor Verification}
\label{sec:monver}
We present our approach to the monitor correctness problem, which reduces checking the existence of missed alarms to policy synthesis on colored MDPs, defined below. The latter problem is well studied. We first formalize the  policy synthesis problem and then discuss the transformation in multiple steps. Here, we focus on showing that there are no missed alarms  of exactly the length of the horizon (an adaption of \Cref{prob:MA}). At the end of the section, we  generalize our construction to finding false alarms and to traces of length \emph{at most} the horizon.


\subsection{Relating Missed Alarms with Color-Consistent Policies}\label{ssec:maintheorem}
Policy synthesis for an MDP for a property $\phi$ thus entails finding a policy for an MDP such that the induced MC entails $\phi$.
A policy for MDP $\mc$ is a function $\sigma\colon \Pi_{fin}^\mc \to \Act$, which selects actions for every finite path. An MDP and a policy induce an MC.
\sj{Moved these sentences down, need to integrate them?}

Colored (Labelled) MDPs are an extension to MDPs that allow expressing dependencies between states that policies must adhere to. The following definition simplifies the notion of colored MDPs from \cite{}, and suffices for our needs:
\begin{definition}[Colored MDP]\label{def:colMDP}
    Given an MDP $\mdp$ with states $\Sts$, a \emph{colored MDP} is a tuple $\mdpc = (\mdp, C, c)$, where $C$ is a set of \emph{colors}, and $c\colon \Sts \to C$ maps states to colors.
\end{definition}
\begin{definition}[Color consistent]
    A memoryless scheduler $\sigma$ for a colored MDP $\mdpc$ is \emph{color consistent}\footnote{Colored MDPs with color-consistent policies coincide with memoryless policies for partially observable MDPs. However, POMDPs often consider history-dependent (belief-based) policies. We use \emph{colored MDPs} to avoid any confusion.} if for states $s, s'$,
    $c(s) = c(s')$ implies $\sigma(s) = \sigma(s')$.
\end{definition}
The set of all color-consistent schedulers is denoted $\ccSched$. Policy synthesis for colored MDPs asks to find a color-consistent scheduler such that the reachability probability to a set of target states is above a certain threshold. Policy synthesis for colored MDPs is NP-hard\cite{}.
%We now present the theorem we focus on in this section.
\begin{theorem}[Main theorem]
    \label{thm:maintheorem}
    Given a HMM $\mc$ with a monitor $\dfa$, threshold $\lambda_s$, horizon $h$, and risk $r$, there is a colored MDP $\mdpc$ with target states $T$ s.t.\
    \[ \exists \sigma \in \ccSched.\ \Prob^{\mdpc}_{\sigma}(\lozenge T) \geq \lambda_s\quad\text{ iff }\quad \exists \tau\in\lang{\mc}.\ \tau \in \missedalarms{\mc,\lambda_s}{= h}\!(\dfa).\]
\end{theorem}
Our proof, outlined in this section, is constructive and we show that we can use the construction to find a $\tau \in \missedalarms{}{}(\dfa)$, whenever such a $\tau$ exists.

We define as \emph{alarm states} $F$ those states which correspond to not accepting states in the DFA. This is equivalent to taking a product with the complement of the monitor.

\begin{definition}[HMM product]
    \label{def:product}
    Given a HMM $\mc=(\Sts, \iota^\mc, \ptrans, Z, \obsf, r)$ and monitor $\dfa=(Q, \Sigma, \delta, \iota^\dfa, F')$, the \emph{product HMM} $\langle\mc_{\times \dfa}, F\rangle$ is the HMM  $\mc_{\times \dfa}                                    = (S \times Q, \langle\iota^\mc,\iota^\dfa\rangle, \ptrans', \obsf', r')$ with $\obsf'(\langle s, q\rangle) = \obsf(s)$,  $r'(\langle s, q\rangle)                              = r(s)$
    \begin{align*}
        \ptrans'(\langle s, q\rangle)(\langle s', q'\rangle) & = \begin{cases}
                                                                     \ptrans(s, s') & \text{if } \delta(q, \obsf(s')) = q' \\
                                                                     0              & \text{otherwise}
                                                                 \end{cases}
    \end{align*}
    And the alarm states $F = S\times F'$.
\end{definition}
In the product, we can find a trace whose conditional trace probability exceeds a threshold iff there is a missing alarm.
We state the decision problem that needs to be solved: It is key to our computational complexity analysis in \Cref{sec:compl}.
\begin{definition}[CTP Decision Problem]
    \label{def:CTPdecprob}
    Given HMM $\mc$ with states~$S$ and risk~$r$, (unary encoded) horizon $h$, alarm states $F\subseteq\Sts$, and threshold $\lambda_s \in \RR^+$,  \[\exists \tau \in \lang{\mc}.~ \sum\nolimits_{\pi\in\Pi^\mc_{=h}\mid\pi_\downarrow \in F}\Prob^\mc(\pi \mid \tau) \cdot r(\pi_\downarrow) \geq \lambda_s.\]
\end{definition}
We denote the set of witnesses $\tau$ to a CTP decision problem as $\mathsf{CTP}(\mc, h, F, \lambda_s)$.
The following lemma states the correctness of the transformation and follows directly from the definition of missing alarms and the product with the complement\sj{Nevertheless, we should have a (slightly) more formal proof}.
\begin{lemma}
    \label{lem:toctp}
    Using the notation from \Cref{thm:maintheorem}, \Cref{def:product} and \Cref{def:CTPdecprob}:
    \[ \exists \tau \in \mathsf{CTP}(\mc_{\times\overline{\dfa}}, h, F, \lambda_s) \quad\text{ iff }\quad \exists \tau \in \missedalarms{\mc,\lambda_s}{= h}\!(\dfa).\]
\end{lemma}

\begin{definition}[Unrolling with risk]
    \label{def:acyclichmm}
    The (risk-)unrolled HMM $\mc_{\blacktriangleright h}$ of a HMM $\mc = \left(\Sts, \init, \ptrans, \Obs, \obsf, r \right)$ with horizon $h$ and alarm states $F \subseteq \Sts$ is the HMM $\mc_{\blacktriangleright h} = (\{1,\hdots,h\} \times \Sts \cup \{ \alarms, \safes, \igs \}, \langle 1,\init\rangle, \ptrans', \Obs \cup \{\en, \ig\}, \obsf')$, with the new transition and observation function:
    \begin{align*}
        \forall_{i\in\{1,\ldots,h-1\}}\quad \ptrans'(\langle i, s\rangle)(\langle i+1, s'\rangle)          & = \ptrans(s)(s')                                                                       \\
        \ptrans'(\langle h, s\rangle)\left(\left\langle h+1, t\right\rangle\right)                         & = \begin{cases}
                                                                                                                   \frac{r(s)}{\max_{s\in\Sts} r(s)}     & \text{if } s \in F \land t = \talrm  \\
                                                                                                                   1 - \frac{r(s)}{\max_{s\in\Sts} r(s)} & \text{if } s \in F \land t = \tsafe  \\
                                                                                                                   1                                     & \text{if } s \notin F \land t = \tig \\
                                                                                                                   0                                     & \text{otherwise}
                                                                                                               \end{cases} \\
        \ptrans'\left(\left\langle h+1, t\right\rangle\right)\left(\left\langle h+1, t\right\rangle\right) & = 1                                                                                    \\
        \obsf'(\langle i, s\rangle)                                                                        & = \obsf(s)                                                                             \\
        \obsf'(\langle h+1, t\rangle)                                                                      & = \begin{cases}
                                                                                                                   \en & \text{if } t \in \{\talrm, \tsafe\} \\
                                                                                                                   \ig & \text{if } t = \tig
                                                                                                               \end{cases}
    \end{align*}
\end{definition}

For Lemma~\ref{lem:toactp}, we must find a trace such that a conditional reachability probability exceeds a threshold. We reformulate this into a policy synthesis problem in MDPs. The transformation combines two ideas: First, in every state, the policy can select the next observation, inspired by~\cite{}. Second, we reformulate conditional reachability probabilities by (standard) regular reachability probabilities, using a construction in \cite{}.
\begin{example}
    We transform the HMM from \cref{fig:actp} to the Colored MDP from \cref{fig:mdpex}. Consider $\actps{2}{i}{2}$ in original the HMM.
    The next observation is either $\dry$ or $\icy$, which corresponds to two new actions for $\actps{2}{i}{2}$ in the colored MDP. The $\dry$ action transitions to the ensuing states with the $\dry$ observation. The remaining probability, $\sfrac{1}{2}$, for the $\dry$ action is directed to the initial state. This is similarly applied to the $\icy$ action.
    State $\actps{2}{c}{2}$ does not reach any $\dry$ states, we still add a $\dry$ action redirecting to the initial state, since we want all states with the same step value to have the same actions.
    State $\actps{3}{d}{1}$ in the HMM only has a transition to an $\ig$ state. We fully remove the $\ig$ state and the $\ig$ observation in the MDP. Finally, we make sure all states with step 3 have the same actions, and thus add the $\en$ action redirecting to the initial state.
\end{example}

\begin{definition}\label{def:polsyntrans}
    Given an acyclic HMM $\mc_{\blacktriangleright h} = \left(\Sts, \init, \ptrans, \Obs, \obsf, r \right)$ as in \Cref{def:acyclichmm}, we define the colored MDP $\mdp_{\gtrdot h} = \left(\left(\Sts', \init, \Act, \ptrans' \right), C, c\right)$ with $\Act = \Obs\setminus\{\ig\}$,
    \begin{align*}
        \ptrans'(\langle i, s\rangle, z)(\langle j, q\rangle) :=
        \begin{cases}
            \ptrans'(\langle i, s\rangle)(\langle j, q\rangle)                                                                              & \text{if } \obsf(\langle j, q\rangle) = z \text{,} \\
            \sum_{(\langle k, q'\rangle)\in S' \mid \obsf(\langle k, q'\rangle) \neq z} \ptrans'(\langle i, s\rangle)(\langle k, q'\rangle) & \text{if } (\langle j, q\rangle) = \iota  \text{,} \\
            0                                                                                                                               & \text{otherwise}
        \end{cases}
    \end{align*}
    $C = \{1,\ldots,h\}$ and $c$ is given such that $c(\langle i,s \rangle) = i$.
\end{definition}
Thus, we transition normally to a state if the action and observation of the target state are equal, otherwise we set the transition probability to zero. All the remaining probability mass is redirected towards the initial state.

\begin{remark}
    All states gain all actions in $\ptrans''$, we can later prune any actions for all states $S^C$ of a color $C$ when these actions only redirect to the initial state from $S^C$.
\end{remark}
The above construction allows for conditioning on a trace $\tau$ by constructing a policy $\sigma$ that selects the $i$th observation of $\tau$ in the state with step $i$.
\begin{definition}[Trace consistent policy]
    Given an MDP $\mdp_{\gtrdot h}$ as defined in \cref{def:polsyntrans}. Together with a trace $\tau \in Z^\star$. A \emph{trace consistent policy} is of the form
    \begin{align*}
        \sigma_\tau(\langle i, s \rangle) = \begin{cases}
                                                \tau^{(i)} & \text{if } i \leq |\tau|, \\
                                                end        & \text{otherwise.}
                                            \end{cases}
    \end{align*}
\end{definition}
Using the coloring as described in \cref{def:polsyntrans}, the trace consistent policies coincide with the color consistent policies. Finding a missed-alarm traces now reduces to solving the following color consistent policy synthesis problem:
\[\exists \sigma\in\ccSched. \Prob^{\sigma}(\lozenge \alarms) > \lambda_s \]
Which is solved using PAYNT \cite{andriushchenkoPAYNTToolInductive2021}.

\begin{lemma}
    \label{lem:tocoloredmdp}
    Given an HMM $\mc$, horizon $h$, alarm states $F$ and threshold $\lambda$:
    \[  \exists \sigma \in \ccSched.  \Prob^{\mdp_{\gtrdot h}}_{\sigma}(\lozenge T) \geq \lambda \quad\text{ iff }\quad \exists \tau \in \lang{\mc_{\blacktriangleright h}}. \sum_{\pi\in\Pi^{\mc_{\blacktriangleright h}}}\Prob^{\mc_{\blacktriangleright h}}(\pi \cdot \talrm \mid \tau \cdot \en) \geq \lambda, \]
\end{lemma}\lm{Fix hbox}

\begin{corollary}
    There exists a map $t(\sigma_\tau) = \tau$ which, given a trace consistent policy $\sigma_\tau$, finds its associated trace $\tau$.
\end{corollary}

\subsubsection{No-False-Alarms Problem}
We modify the transformation from \cref{thm:maintheorem} such that it solves the no-false-alarms problem. The no-false-alarms problem differs in two ways from the no-missed-alarms problem. Fristly, instead of finding a trace not in the monitor, we have to find a trace in the monitor. Secondly, this trace should have a conditional probability below the threshold. We solve these one at a time after stating \cref{thm:maintheorem} for no-false-alarms.

\begin{theorem}
    \label{thm:nfatheorem}
    Given a HMM $\mc$ with a monitor $\dfa$, threshold $\lambda_u$, horizon $h$, and risk $r$, there exists a colored MDP $\mdpc$ and a $\lambda$ such that
    \[ \exists \sigma \in \ccSched. \Prob^{\mdpc}_{\sigma}(\lozenge T) \leq \lambda \quad\text{ iff }\quad \exists \tau \in \falsealarms{\mc,\lambda_s}{= h}\!(\dfa).\]
\end{theorem}
\begin{proof}
    We reuse the transformation from \cref{thm:maintheorem} with the following modifications. We no longer take the complement of the monitor. Instead of doing reachability on the alarm state, we instead do reachability on the safe state.
\end{proof}

\paragraph{MAT framework.}
We briefly recap the MAT framework, for details see~\cite{DBLP:journals/cacm/Vaandrager17}.
A minimally adequate teacher answers two types of questions:
A membership query (MQ), which in our setup means \emph{should a trace be accepted by the monitor?}, and an equivalence query (EQ), \emph{is this monitor correct?}
Furthermore, if the answer to an equivalence query is negative, we must provide a counterexample that witnesses why the monitor is not correct.
Various algorithms implementing the MAT framework for DFA learning exist. For the purpose of this paper, we use the L$^{\star}$ algorithm \cite{DBLP:journals/iandc/Angluin87} to learn a monitor. The learner asks MQs to the teacher until a hypothesis monitor can be constructed which is consistent with the MQs. Once such a hypothesis is constructed, the monitor is verified using an EQ.

\paragraph{Conformance queries.}
An alternative to the EQ in \cref{def:toverEQ} is a conformance query \cite{groceAdaptiveModelChecking2002}. It tests a monitor by sampling traces from the HMM and checking if the MQ and the monitor agree. If all the monitor and the MQ don't agree on a trace, it is given as a counter example. In our approach we use a hybrid of the two EQs. Monitors produced early in the learning process often contain many missed alarms and false alarms. Verification can find them, however, applying the transformation from \cref{sec:monver} has a constant cost. Conformance queries can often find a counter example faster if one has a high probability of occurring.

\section{Computational Complexity}\label{sec:compl}
\begin{theorem}
    \label{lem:complexity}
    \emph{Is a monitor correct?} (w.\ unary coded horizon) is coNP-complete.
\end{theorem}
In fact, we study the dual to this problem, i.e., checking the existence of a counterexample. We call this problem \emph{monitor co-verification}. For monitor co-verification, \emph{membership} in NP follows as  false alarms or a missed alarm (of length up to horizon, i.e., polynomially sized) are witnesses.
Verifying whether a trace is a false or missed alarm can be done in polynomial time, by checking whether the automaton accepts it and computing the trace risk (see \cref{sec:learning}).

To establish NP-hardness, we consider the CTP problem from~\Cref{def:CTPdecprob}. As a solution to the monitor co-verification problem solves the CTP problem (using a trivial monitor), this implies NP-hardness of the former problem.
\begin{lemma}
    \label{lem:ctpdecnphard}
    The CTP Decision Problem is NP-hard.
\end{lemma}
The lemma holds even for $\lambda = 1$ and acyclic HMMs. We reduce the problem  from 3CNF. We outline the construction below.

Consider a 3CNF formula $\varphi = \bigwedge c_1 \dots c_m$ over variables $X$, $|X|=n$, with clause $c_j = \ell^1_j \lor \ell^2_j \lor \ell^3_j$ and each literal $\ell^i_j \in \{ x, \neg x \mid x \in X\}$. We transform this into a CTP instance with $\lambda = 1$ and an acyclic HMM $\mc_\varphi$  with observations $\Obs= \{ \#, \bot, \top \}$. The only state with positive risk  is a dedicated state $t$ with $r(t) = 1$. The crux of the construction is that there is a trace with risk $1$ iff there is a satisfying assignment to $\varphi$. In the constructed HMM, there is a trace with risk $1$ iff there is a trace where all corresponding\sj{make sure that corresponding paths are the right formal notion} paths end in state $t$.

Before we give a formal definition of $\mc_\varphi$, we give some intuition.
We represent assignments $\alpha\colon X \rightarrow \{\bot,\top \}$ by traces through the HMM $\#\# \cdot \alpha(x_0) \cdot \# \cdot \alpha(x_1) \dots \cdot \#$. We create gadgets for every clause. The gadgets are connected as in \Cref{fig:gadgetsconnected}: That is, intuitively, $\mc_\varphi$ randomly selects a clause $c_j$ with probability $\nicefrac{1}{m}$ and transitions into gadget $\mathcal{G}_j$ defined below. Next, we show that the gadget reaches positive risk in every gadget only if the corresponding clause is satisfied by the assignment.

The gadget $\mathcal{G}_j$ intuitively `evaluates' $c_j$ with respect to an assignment $\alpha$, by starting in $s_{1,j}$ with a gadget for this clause, as exemplified in Fig.~\ref{fig:hmmgadget}.  We enter the gadget with trace $\# \cdot \alpha(x_0) \cdot \# \cdot \alpha(x_1) \dots \cdot \#$, i.e., the first observation has been matched by the initial state.
A path (or its trace) through $\mathcal{G}_j$ `reads' variable $x_i$ in state $s_{i,j}$ and transitions to $s_{i,j}^\top$ or to $s_{i,j}^\bot$. However, only one of these paths corresponds to the trace. Thus, in every gadget, there is only one path that corresponds to a given trace and so they can be used interchangeably.
State $s_{i,j}$, $s_{i,j}^\top$ and $s_{i,j}^\bot$ have observations $\#, \top, \bot$, respectively. We note that a path  reaches $s_{i,j}^\top$ if $\alpha(x_i) = \top$ and (analogously for $\bot$).
From there onwards,  if the literal $x_i$ (or $\neg x_i$) occurs in clause $c_j$, we transition from $s_{i,j}^\top$ (or $s_{i,j}^\bot$, resp.) to $s_{i+1,j}'$, otherwise, we transition to $s_{i+1,j}$. A path ends in $t$ only via a state $s'_{i,j}$, which is only possible if the path visits a state corresponding to a literal in the clause. Thus, (conditionally) reaching state $s'_{i,j}$ with positive conditional probability means that the partial assignment $\alpha(x_1),\dots,\alpha(x_i)$ satisfies clause $c_j$, while reaching $s_{i,j}$ with positive conditional probability means that the clause is either unresolved or unsatisfied given the partial assignment.

\end{itemize}

\subsubsection{Approximability}

Approximating the verification of monitors is also hard, in the sense that it is hard to approximate the maximal risk that a monitor admits.

The maximal risk problem is an optimization problem
\begin{definition}[CTP Optimization Problem]
    Given an HMM $\mc$ with states in $S$, a unary encoded horizon $h$, a set of alarm states $A\subseteq\Sts$: \[\max_{\tau \in \lang{\mc}}~ \sum_{\pi\in\Pi^\mc_{=h}\mid\pi_\downarrow \in A}\Prob^\mc(\pi \mid \tau) \cdot r(\pi_\downarrow)\]\sj{check notation for paths of length $h$}
\end{definition}\sj{Sync with CTP definition}
\begin{lemma}
    \label{lem:ctpapxhard}
    The CTP Optimization Problem is APX-hard.
\end{lemma}
This follows from a strict reduction from MAX-3SAT, which is an inapproximable and APX-hard problem~\cite{DBLP:journals/jacm/Hastad01}. The construction coincides with the reduction in \Cref{lem:ctpdecnphard} by observing that the conditional probability to reach a $t$ state is given by $\nicefrac{1}{m}$ times the number of satisfied clauses, i.e., we can compute the maximal number of satisfied clauses in $\varphi$ on the HMM $\mc_\varphi$.

\paragraph{Are the monitors correct?}
Using a proper EQ, we always learn correct monitors. We validate this experimentally \emph{and} show that the baseline does not always yield correct monitors. Therefore, for any monitor we determine  the unsafe trace with the lowest risk (\emph{actual alarm threshold}, $\lambda_u^{\min}$) and the safe trace with the highest risk (\emph{actual no-alarm threshold}, $\lambda_s^{\max}$). In a correct monitor, we have $\lambda_u^{\min} \geq \lambda_u$ and $\lambda_s^{\max} \leq \lambda_s$.
\Cref{fig:learnThrToverS,fig:learnThrToverNS} together and \cref{fig:learnThrSampling} show $\lambda_u^{\min}$ and $\lambda_s^{\max}$ for \alg learning and baseline learning respectively. A monitor is correct if its red bar never touches the green area, and the green bar never touches the red area. In 6 out of 38 benchmarks the baseline learns a monitor that misses alarms. No monitors had false alarms.
