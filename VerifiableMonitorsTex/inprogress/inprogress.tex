\begin{abstract}
    Runtime monitors for hidden Markov models obtain a sequence of observations and must classify whether the system is currently deemed safe. 
    A system is deemed safe, if the probability of being in a dangerous state conditioned on these observations exceeds some acceptable threshold.
    The key contribution of this paper is the first correct-by-construction synthesis method for such monitors, represented as finite state machines.
    First, we establish the coNP-hardness of checking whether a finite state machine  is indeed a correct monitor, i.e., a monitor without misclassifications.
    Second, we provide a reduction that reformulates the search for misclassifications into a standard probabilistic system synthesis problem.
    Third, we integrate the verification routine into an active automata learning routine to synthesise correct monitors.
    Fourth, we provide a prototypical implementation that shows the feasibility and limitations of the approach on a series of benchmarks.

\section{Introduction}
Runtime assurance is an essential ingredient in the deployment of safe autonomous systems.
Runtime monitors provide assurance by flagging potentially dangerous system behavior by observing a system execution.
More precisely, a monitor receives a stream of observations about the system and outputs a verdict, e.g., it raises an alarm that the system has left some safety envelope. A monitor is correct, if it correctly raises such alarms based on a formal specification.
Various challenges in creating correct runtime monitors for (semi-)autonomous systems have been identified~\cite{}, such as:
(1)~the state of the system is only partially observable, i.e., the stream of observations comes from sensor readings and does uniquely identify the state of a system,
(2)~the behavior of the system and/or the sensors may be subject to probabilistic uncertainty,
(3)~the monitor itself is subject to resource constraints (time, memory, etc), and
(4)~the monitor is itself safety-critical and should therefore be subject to extensive validation.
Challenges (1,2) can be (partially) addressed  by modelling the system as a hidden Markov model (HMM), Challenges (3,4) can be (partially) addressed by representing a monitor as, e.g., a (small) finite state machine.
Concretely, this paper focuses on the following question:
\emph{Is a given finite state machine a correct monitor for a given HMM?}
The paper studies the complexity of this problem, provides a practical verification procedure, and embeds this verification procedure into a learning framework.


\paragraph{What are HMMs?}
In this paper, we assume that the system including its sensors is adequately described as a discrete Hidden Markov Model (HMM).
Markov chains (MCs) describe system behavior subject to probabilistic uncertainty.
Paths through an MC are sequences of states that describe system executions.
% MCs can be seen as specifying a distribution over these paths. 
HMMs extend MCs by labelling their states with colours (aka \emph{observations}).
Intuitively, the observations can be used to model the information that the monitor receives.
In HMMs, every path can be lifted to a sequence of observations, which we call a \emph{trace}.
The trace associated to a system execution describes the information received by the monitor.


% where every state is labelled with a distribution over observations, see Fig.~\ref{}. Paths are sequences of states and MCs can be seen as specifying a distribution over these paths. An HMM extends MCs by assigning to every state a colour (an \emph{observation}). A trace is a sequence of these observations. 

\paragraph{Risk of a trace.}
In runtime monitoring, an essential task is to estimate whether the current system is dangerous. In the simplest setting, a specification describes a set of states is dangerous. The risk of a trace is then the probability of ending in a dangerous state, conditioned on the fact that the system execution matches the trace.
For a given trace, we may compute this risk, e.g., either via a reduction to model checking~\cite{} or by a (forward) filtering that tracks a distribution over the current states~\cite{}.


\paragraph{What are correct monitors?}
Often, we are not interested in this precise risk, but rather want to  partition the set of horizon-bounded traces into safe and unsafe traces:
Unsafe traces are traces whose risk does exceed some fixed \emph{acceptable threshold}, all other traces are safe.
Indeed, a \emph{correct} monitor obtains a trace and raises an alarm iff a trace is not safe. The key verification problem in this paper asks whether a monitor, represented as a deterministic finite automaton, accepts (i.e., raises an alarm) exactly the unsafe traces. We note that in this paper, we only consider traces that are bounded by some fixed horizon.

\paragraph{Novel complexity results.}
Deciding whether a monitor is correct is coNP-complete (\Cref{lem:complexity}).
In particular, we study the dual problem that asks whether a monitor misclassifies at least one trace.
We consider a simpler variant of this problem that asks whether there is a bounded trace in the HMM that is unsafe. \footnote{The results also apply to checking for safe traces.}
The problem is clearly in NP (assuming that the bound is unary encoded).
We show that this simplified problem is also strongly NP-hard (\Cref{lem:ctpdecnphard}) and the corresponding optimization problem, that asks to approximate the risk of the trace with the heighest risk, is APX-hard (\Cref{lem:ctpapxhard}).
Intuitively, APX-hardness means that the existence of an efficient approximation algorithm would have major complexity-theoretic implications.


\paragraph{Our verification approach.}
While deciding whether a monitor is correct is thus in general not tractable, we suggest to utilise recent advances in the synthesis of probabilistic systems.
We concentrate on proving the absence of missed alarms, i.e., we concentrate on showing that the monitor correctly identifies every unsafe trace. A similar reduction works for showing the absence of spurious alarm.
First, for a single trace, computing the risk can be reduced to computing a reachability probability in a MC that is intuitively a kind of product between a DFA that accepts exactly the trace and the HMM~\cite{}.
Thus, it is natural to reduce our problem to the question: Is there a DFA (accepting exactly one trace, accepted by our monitor)
such that the probability in the product MC exceeds a threshold.
If the answer is no, then the monitor has no missed alarms.
This decision problem can be solved using (exact) probabilistic model synthesis methods~\cite{} as implemented in the tool Paynt~\cite{}\footnote{Alternatively, in Section~\ref{sec:useparametersynthesis}, we show that we can construct a product of an acyclic DFA accepting sets of traces and the HMM. However, this yields partially observable MDP, where we must search for memoryless policies, similar to a construction in~\cite{DBLP:conf/tacas/BadingsVJSJ24}. This problem is equivalent to the synthesis problem and can therefore also be solved using Paynt~\cite{DBLP:conf/uai/Andriushchenko022}.}. We discuss our verification approach in \Cref{sec:monver}.

\paragraph{Our learning approach. }
Ultimately, we do not only want to verify the correctness of a monitor, but we want to construct such a monitor.
Above, we already mentioned that the monitors are formal languages. When considering bounded trace, these monitors are regular, and can thus be captured using DFAs.
Inspired by a recent short paper~\cite{}, we investigate learning these monitors:
We choose to do this using active automata learning~\cite{DBLP:journals/iandc/Angluin87,DBLP:journals/cacm/Vaandrager17}, which closely relates to other popular oracle-guided inductive synthesis loops~\cite{}.
To use active automata learning (AAL),
we must provide a \emph{membership oracle} that decides for a single trace whether the risk of the trace exceeds a threshold and an \emph{equivalence oracle} that decides whether a given hypothesis monitor is indeed correct. The membership oracle can be implemented by computing the risk for a single trace as above and comparing it to a threshold, while the equivalence oracle can be implemented by checking for the absence of missed and false alarms.
Practically, some minor modifications are necessary to embed our approach in  AAL framework: We have to handle the finite horizon, resolve some traces that are not relevant for the monitor, and we combine conformance and equivalence queries to boost performance.

\paragraph{Contributions.}
In summary, this paper provides a first framework to learn runtime monitors --- encoded as finite automata --- that are verified to be correct with respect to a given HMM. The main contributions are as follows:
(1)~We study monitor verification and show complexity-theoretic hardness of the underlying problem, even in an approximative setting.
(2)~We reduce the monitor verification to probabilistic model synthesis problems that exhaustively search for a counterexample.
(3)~We learn monitors by using the verifier above to answer equivalence queries in conjunction with active automata learning.
(4)~We demonstrate the feasibility and limitations of both the verifier and the learner on a series of benchmarks. Our prototype can handle HMMs with thousands of states, up to a hundred observations, and monitors with over 100 states.

\section{Problem Statement}
In this section, we recap standard notions on stochastic models and finite automata and introduce the formal problem statement.

\paragraph{Notation.}
Let $X$ be a finite set. A distribution $\mu$ over $X$ is a mapping $\mu\colon X \rightarrow [0,1]$ such that $\sum_{x \in X} \mu(x) = 1$. The set of all distributions over $X$ is written $\Distr(X)$.


\subsection{Models}
In this section, we briefly introduce MDPs, which are important later, and HMMs which are crucial to the problem statement. Details can be found in \cite{baierPrinciplesModelChecking2008}.


\begin{definition}[MDP]
    A \emph{Markov decision process} (MDP) is a tuple $\mdp = \left(\Sts, \init, \Act, \ptrans \right)$ with a  countable nonempty set  $\Sts$ of \emph{states},  the \emph{initial state} $\init \in \Sts$, the \emph{partial transition function} $\ptrans\colon \Sts \times \Act \rightharpoonup \Distr(\Sts)$.
\end{definition}
We use $\Act(s) = \{ \act \mid \ptrans(s, \act) \neq \bot \}$ as the set of enabled actions. We assume that there are no deadlocks, i.e., for every $s \in \Sts$, $\Act(s) \neq \emptyset$.
A path $\pi$ in an MDP is a sequence $s_0\act_0s_1\hdots \in (\Sts \times \Act)^{*} \times \Sts$, such that $a_i \in \Act(s_i)$ and $P(s_i, a_i)(s_{i+1}) > 0$ for every $i \geq 0$. %The set of all infinite paths is $\Pi^\mc$. 
The set of (finite) paths is denoted by $\Pi_{fin}^\mdp$, the set of paths of at most length $h$ are $\Pi_h^\mdp$, and the set of paths of exactly length $h$ are $\Pi_{=h}^\mdp$. If the MDP is clear from the context, the $\mdp$ is dropped.

A policy for an MDP is a function $\sigma\colon \Pi^\mc \to \Act$, which given a path through the MDP, gives a action. Given a policy, a MDP can be induced into a MC. Policy synthesis for an MDP for a property $\phi$ thus entails finding a policy for an MDP such that the induced MC entails $\phi$.

A \emph{Markov chain} (MC) is an MDP where $|\Act(s)| = 1$ for every state $s \in \Sts$. We simplify notation and write MCs also as tuple $\left(\Sts, \init,  \ptrans \right)$ and write $\ptrans(s)$ to refer to the unique distribution $\ptrans(s, \act)$.  Paths in a MC can be simplified to be sequences of (only) states.

\begin{definition}
    A deterministic finite automaton (DFA) is a tuple $\dfa=\left(\StsD, \alphabet, \trans, \init, \final\right)$. $\StsD$ is a finite set of states, $\alphabet$ is an alphabet, $\trans\colon \StsD \times \alphabet \rightharpoonup \StsD$ is a (partial) transition function, $\init$ is the initial state, and $\final \subseteq \StsD$ is a set of accept (or: final) states.
\end{definition}
Let $\varepsilon$ denote the empty word. We lift the transition relation to words, which is defined recursively: $\trans^*(q, \varepsilon) = \varepsilon$ if $a \in \alphabet$ and $\trans^*(q, w \cdot a) = \trans(\trans^*(q,w), a)$.

The \emph{language} $\lang{\dfa}$ of a DFA $\dfa$ is the set of all words accepted by the $\dfa$, $\lang{\dfa} = \{ w \in \alphabet^* \mid \trans^*(\init, w) \in \final \}$.

\subsection{Formal Problem Statement}
The monitor verification problem consists of two sub problems. The no false alarms problem, and the no missed alarms problem. Both of these sub problems entail finding the trace with either the minimum or maximum risk. We will discuss the missed alarm problem in the following sections. In \cref{ssec:nfa} we will adapt to the no false alarms problem.

The no missed alarms problem entails ensuring that all traces from the HMM, which are not in the monitor, have at most a risk of $\lambda$.

\begin{definition}[Risk of a trace]
    Given a HMM $\mc$. The risk of trace $\tau \in \lang{\mc}$ is: \[ R_r(\tau) \defeq \sum_{\pi\in\Pi^h_\mc}\Prob(\pi \ |\ \tau) \cdot r(\pi_{\downarrow}).\]
\end{definition}
The risk of a trace can be used to partition the set of all traces.
\begin{definition}[Safe traces/Unsafe traces]
    Given a HMM $\mc$, a \emph{safe threshold} $\lambda_s \in \mathbb{R}$ and an \emph{unsafe threshold} $\lambda_u \in \mathbb{R}$ with $\lambda_s \leq \lambda_u$.
    A trace $\tau \in \lang{\mc}$ with $R(\tau) > \lambda_u$ is an \emph{unsafe trace} (for $\mc$ and $\lambda_u$). Conversely, trace $\tau$ is a \emph{safe trace}, if $R(\tau) < \lambda_s$ (for $\mc$ and $\lambda_u$).
\end{definition}
In the definition above, we deliberately do not require $\lambda_s = \lambda_u$. In particular, by picking $\lambda_s < \lambda_u$, some traces are neither safe nor unsafe. We write $\safetraces{\mc,\lambda_s}{\leq h}$ to denote the set of safe traces of length at most $h$ and $\unsafetraces{\mc,\lambda_u}{\leq h}$ for the set of unsafe traces of length at most $h$. We often omit the sub/superscripts.

We are now ready to define correct monitors.
\begin{definition}[Monitor]
    A DFA $\dfa$ is a monitor for HMM $\mc$ if the alphabet for $\dfa$ coincides with the observations in $\mc$.
\end{definition}
\begin{definition}[Missed/False alarms]
    Given a monitor $\dfa$ for HMM $\mc$, a horizon $h$, and thresholds $\lambda_s \leq \lambda_u \in \mathbb{R}$, the set of \emph{missed alarms} is $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa) \defeq \unsafetraces{\mc,\lambda_u}{\leq h} \setminus \lang{\dfa}$. The set of \emph{false alarms} is $\falsealarms{\mc,\lambda_s}{\leq h}\!(\dfa) \defeq \safetraces{\mc,\lambda_s}{\leq h} \cap \lang{\dfa}$.
    % a safe trace $\tau \in \Obs^{\leq h}$ with $R(\tau) > \lambda_s$ and $\tau \not\in \lang{\dfa}$ is a \emph{missed alarm}. Conversely, trace $\tau$ is a \emph{false alarm}, if $R(\tau) < \lambda_u$ and $\tau \in \lang{\dfa}$. 
\end{definition}
We often drop the sub/superscripts. In terms of a classification problem, missed alarms are false negatives, while false alarms are false positives.
\begin{definition}[Correct monitor]
    Given thresholds $\lambda_s,\lambda_u$ and horizon $h$, a monitor $\dfa$ for $\mc$ is \emph{correct} if $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa) = \emptyset = \falsealarms{\mc,\lambda_u}{\leq h}\!(\dfa)$.
\end{definition}
A correct monitor raises an alarm for all unsafe traces and for no safe trace:
\begin{corollary}%[Correct monitor]
    A monitor $\dfa$ is correct iff $ \unsafetraces{\mc,\lambda_u}{\leq h} \subseteq \lang{\dfa} \subseteq \alphabet^* \setminus \safetraces{\mc, \lambda_s}{\leq h}$.% is a correct monitor.
\end{corollary}

\begin{mdframed}\textbf{Problem statements.} This paper tackles the following problem statements:
    Given HMM $\mc$, thresholds $\lambda_s,\lambda_u$ and horizon $h$,
    \begin{enumerate}
        \item Given monitor $\dfa$ for $\mc$, are there missed alarms, i.e., is $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa)\! = \emptyset$?
        \item Given monitor $\dfa$ for $\mc$, are there false alarms, i.e., is $\falsealarms{\mc,\lambda_u}{\leq h}\!(\dfa)\! = \emptyset$?
        \item Find a correct monitor $\dfa$ for $\mc$ w.r.t.\ $\lambda_s, \lambda_u$ and $h$.
    \end{enumerate}
\end{mdframed}
Problem statements 1 and 2 together allow checking whether a monitor is correct.
\begin{remark}
    A correct monitor must exist, as $\unsafetraces{}{\leq h}$ is finite and thus regular.
\end{remark}

\section{Monitor Verification}
\label{sec:monver}
In this section we present our approach to the monitor correctness problem, which essentially reduces checking the existence of missed alarms to a policy synthesis problem on colored MDPs. The latter is well studied. We first formalize this policy synthesis problem and then discuss the transformation in multiple steps.
In our presentation, we focus on the no missed alarms problem with traces of exactly the length of the horizon. At the end of the section, we  generalize our construction to both the no false alarms problem and traces of length at most the horizon.


\subsection{Relating Missed Alarms with Color-Consistent Policies}\label{ssec:maintheorem}
Colored (Labelled) MDPs are a mild extension to MDPs that allow expressing dependencies between states that policies must adhere to. The following definition simplified the notion of colored MDPs from \cite{} suffices for our needs:
\begin{definition}[Colored MDP]
    Given an MDP $\mdp$ with states $\Sts$, a \emph{colored MDP} is a tuple $\mdpc = (\mdp, C, c)$, where $C$ is a set of \emph{colors}, and $c\colon \Sts \to C$ maps states to colors.
\end{definition}
\begin{definition}[Color-consistent scheduler]
    A memoryless scheduler $\sigma$ for a colored MDP $\mdpc$ is \emph{color consistent}\footnote{Colored MDPs with color-consistent policies coincide with memoryless policies for partially observable MDPs. However, POMDPs often consider history-dependent (belief-based) policies. We use colored MDPs to avoid any confusion.} if for every two states $s$ and $s'$
    \[ c(s) = c(s') \quad\text{ implies }\quad  \sigma(s) = \sigma(s'). \]
\end{definition}
The set of all color-consistent schedulers is denoted $\ccSched$. Policy synthesis for colored MDPs asks to find a color consistent policy such that the induced reachability probability is above (or below) a certain threshold. Policy synthesis for colored MDPs is NP-hard.
%We now present the theorem we focus on in this section.
\begin{theorem}[Main theorem]
    \label{thm:maintheorem}
    Given a HMM $\mc$ with a monitor $\dfa$, threshold $\lambda_s$, horizon $h$, and risk $r$, there exists a colored MDP $\mdpc$ such that
    \[ \exists \sigma \in \ccSched. \Prob^{\mdpc}_{\sigma}(\lozenge T) \geq \lambda_s\quad\text{ iff }\quad \exists \tau \in \missedalarms{\mc,\lambda_s}{= h}\!(\dfa).\]
\end{theorem}\lm{Where does T come from? Add to the start of the theorem?}
Our proof is constructive and we show that we can use the construction to find a $\tau \in \missedalarms{}{}(\dfa)$, whenever such a $\tau$ exists.

\subsubsection{CTP problem}


Before we combine the HMM and the monitor, we have to modify the monitor. Since we are looking for missed alarms, we are interested in all traces that are not in the monitor. We thus take the complement of the monitor.
Next, we take (a mild variation of) a standard product construction~\cite[Section~10.3]{baierPrinciplesModelChecking2008} between HMM and DFA.  We also keep track of all resulting states for which the complement monitor accepts in a set of alarms states $F$ of the product HMM.
\begin{example}
    \Cref{fig:ctphmm} shows the product of the HMM and monitor $\dfaB$ from \Cref{fig:centralexample}. In particular, when starting in the initial states $d$ and $1$, respectively, the HMM can transition to the $i$ state with probability $\sfrac{9}{10}$. This is an $\icy$ state, and thus the monitor takes the $\icy$ transition to the $2$ state. This results in the new product state of $i$ and $2$, with a transition from $\langle d, 1\rangle$ to $\langle i, 2\rangle$ with probability $\sfrac{9}{10}$.

    The set of Alarm states are all the states in monitor $\dfaB$ which do not accept. Thus they are the initial $\langle d, 1\rangle$ state and any product states where the monitor is in state $3$.
\end{example}
\begin{definition}[HMM product]
    \label{def:product}
    Given a HMM $\mc=(\Sts, \iota^\mc, \ptrans, Z, \obsf, r)$ and a monitor for $\mc$, $\dfa=(Q, \Sigma, \delta, \iota^\dfa, F')$. The product HMM $\mc_{\times \dfa}$ is the HMM:
    \begin{align*}
        \mc_{\times \dfa}                                    & = (S \times Q, \langle\iota^\mc,\iota^\dfa\rangle, \ptrans', \obsf', r') \\
        \ptrans'(\langle s, q\rangle)(\langle s', q'\rangle) & = \begin{cases}
                                                                     \ptrans(s, s') & \text{if } \delta(q, \obsf(s')) = q' \\
                                                                     0              & \text{otherwise}
                                                                 \end{cases}    \\
        \obsf'(\langle s, q\rangle)                          & = \obsf(s)                                                               \\
        r'(\langle s, q\rangle)                              & = r(s)
    \end{align*}
    The alarm states become:
    \begin{align*}
        F = S\times F'
    \end{align*}
\end{definition}
In the product, we can find a trace whose conditional trace probability exceeds a threshold iff there is a missing alarm.
We state the decision problem that needs to be solved on the product HMM as it is key to our computational complexity analysis in \Cref{sec:compl}.
\begin{definition}[CTP Decision Problem]
    \label{def:CTPdecprob}
    Given a HMM $\mc$ with states in $S$, a unary encoded horizon $h$, a set of alarm states $F\subseteq\Sts$, a risk function $r$, and a threshold $\lambda_s \in (0,1]$,  \[\exists \tau \in \lang{\mc}.~ \sum_{\pi\in\Pi^\mc_{=h}\mid\pi_\downarrow \in A}\Prob^\mc(\pi \mid \tau) \cdot r(\pi_\downarrow) \geq \lambda.\]\textsl{}
\end{definition}
We denote the set of witnesses $\tau$ to a CTP decision problem as $\mathsf{CTP}(\mc, h, F, \lambda_s)$.
The following lemma states the correctness of the transformation and follows directly from the definition of missing alarms and the product with the complement \sj{Nevertheless, we should have a (slightly) more formal proof}.
\begin{lemma}
    \label{lem:toctp}
    Using the notation from \Cref{thm:maintheorem}, \Cref{def:product} and \Cref{def:CTPdecprob}:
    \[ \exists \tau \in \mathsf{CTP}(\mc_{\times\overline{\dfa}}, h, F, \lambda_s) \quad\text{ iff }\quad \exists \tau \in \missedalarms{\mc,\lambda_s}{= h}\!(\dfa).\]
\end{lemma}

\begin{definition}\label{def:polsyntrans}
    Given an acyclic HMM $\mc_{\blacktriangleright h} = \left(\Sts, \init, \ptrans, \Obs, \obsf, r \right)$ as in \Cref{def:acyclichmm}, we define the colored MDP $\mdp_{\gtrdot h} = \left(\left(\Sts', \init, \Act, \ptrans' \right), C, c\right)$ with $\Act = \Obs\setminus\{\ig\}$,
    \begin{align*}
        \ptrans'(\langle i, s\rangle, z)(\langle j, q\rangle) :=
        \begin{cases}
            \ptrans'(\langle i, s\rangle)(\langle j, q\rangle)                                                                             & \text{if } \obsf(\langle j, q\rangle) = z \\
            \sum_{(\langle k, q'\rangle)\in S'\ |\ \obsf(\langle k, q'\rangle) \neq z} \ptrans'(\langle i, s\rangle)(\langle k, q'\rangle) & \text{if } (\langle j, q\rangle) = \iota  \\
            0                                                                                                                              & \text{otherwise}
        \end{cases}
    \end{align*}
    $C = \{1,\ldots,h\}$ and $c$ is given such that $c(\langle i,s \rangle) = i$.
\end{definition}

\subsubsection{No False Alarms Problem}
We modify the transformation from \cref{thm:maintheorem} such that it solves the no false alarms problem. The no false alarms problem differs in two ways from the no missed alarms problem. Fristly, instead of finding a trace not in the monitor, we have to find a trace in the monitor. Secondly, this trace should have a conditional probability below the threshold. We solve these one at a time after stating \cref{thm:maintheorem} for no false alarms.

\begin{theorem}
    \label{thm:nfatheorem}
    Given a HMM $\mc$ with a monitor $\dfa$, threshold $\lambda_u$, horizon $h$, and risk $r$, there exists a colored MDP $\mdpc$ and a $\lambda$ such that
    \[ \exists \sigma \in \ccSched. \Prob^{\mdpc}_{\sigma}(\lozenge T) \leq \lambda \quad\text{ iff }\quad \exists \tau \in \falsealarms{\mc,\lambda_s}{= h}\!(\dfa).\]
\end{theorem}
\begin{proof}
    We reuse the transformation from \cref{thm:maintheorem} with the following modifications. We no longer take the complement of the monitor. Instead of doing reachability on the alarm state, we instead do reachability on the safe state.
\end{proof}

We briefly recap the MAT framework, for details see~\cite{DBLP:journals/cacm/Vaandrager17}.
A minimally adequate teacher is a teacher answer two types of questions:
A membership query, which in our setup means \emph{should a trace be part of the monitor?} and an equivalence query, \emph{is this monitor correct?}

Various algorithms implementing the MAT framework for DFA learning exist. For the purpose of this paper, we use the L$^{\star}$ algorithm \cite{DBLP:journals/iandc/Angluin87}. Roughly, the learner will ask the teacher membership queries to construct a hypothesis, i.e., some monitor that is consistent with the answers from the teacher. Once such a hypothesis is constructed, the monitor is verified using an equivalence query.

\section{Computational Complexity}\label{sec:compl}
\begin{theorem}
    \label{lem:complexity}
    Checking whether a monitor is correct given a unary encoded horizon is coNP-complete.
\end{theorem}
In fact, we study in this section the dual to this problem, i.e., checking the existence of a counterexample. We call this problem \emph{monitor co-verification}. For monitor co-verification, \emph{membership} in NP follows as false alarms or a missed alarm (of length up to horizon) are witnesses.
Verifying whether a trace is indeed a false alarm can be done in polynomial time by first checking whether the automaton accepts it and then computing the trace risk and comparing it to the thresholds.
Each of these steps can be done in polynomial time~\cite{}.  \sj{Maybe this result is so crucial that we should explain somewhere how this works in polynomial time. I guess it also follows from the construciton in section 3 if we restrict the monitor to just contain one trace?}

To establish NP-hardness, we consider the CTP problem from~\Cref{def:CTPdecprob}. As a solution to the monitor co-verification problem suffices to solve the CTP problem (using a trivial monitor), this implies NP hardness of the former.
\begin{lemma}
    \label{lem:ctpdecnphard}
    The CTP Decision Problem is NP-hard.
\end{lemma}
The lemma holds even for $\lambda = 1$ and acyclic HMMs. We reduce the problem  from 3CNF. We outline the construction below.

Consider a 3CNF formula $\varphi = \bigwedge c_1 \dots c_m$ over variables $X$, $|X|=n$, with clause $c_j = \ell^1_j \lor \ell^2_j \lor \ell^3_j$ and each literal $\ell^i_j \in \{ x, \neg x \mid x \in X\}$. We transform this into a CTP instance with $\lambda = 1$ and an acyclic HMM $\mc_\varphi$  with observations $\Obs= \{ \#, \bot, \top \}$. The only state with positive risk  is a dedicated state $t$ with $r(t) = 1$. The crux of the construction is that there is a trace with risk $1$ iff there is a satisfying assignment to $\varphi$. In the constructed HMM, there is a trace with risk $1$ iff there is a trace where all corresponding\sj{make sure that corresponding paths are the right formal notion} paths end in state $t$.

Before we give a formal definition of $\mc_\varphi$, we give some intuition.
We represent assignments $\alpha\colon X \rightarrow \{\bot,\top \}$ by traces through the HMM $\#\# \cdot \alpha(x_0) \cdot \# \cdot \alpha(x_1) \dots \cdot \#$. We create gadgets for every clause. The gadgets are connected as in \Cref{fig:gadgetsconnected}: That is, intuitively, $\mc_\varphi$ randomly selects a clause $c_j$ with probability $\nicefrac{1}{m}$ and transitions into gadget $\mathcal{G}_j$ defined below. Next, we show that the gadget reaches positive risk in every gadget only if the corresponding clause is satisfied by the assignment.

The gadget $\mathcal{G}_j$ intuitively `evaluates' $c_j$ with respect to an assignment $\alpha$, by starting in $s_{1,j}$ with a gadget for this clause, as exemplified in Fig.~\ref{fig:hmmgadget}.  We enter the gadget with trace $\# \cdot \alpha(x_0) \cdot \# \cdot \alpha(x_1) \dots \cdot \#$, i.e., the first observation has been matched by the initial state.
A path (or its trace) through $\mathcal{G}_j$ `reads' variable $x_i$ in state $s_{i,j}$ and transitions to $s_{i,j}^\top$ or to $s_{i,j}^\bot$. However, only one of these paths corresponds to the trace. Thus, in every gadget, there is only one path that corresponds to a given trace and so they can be used interchangeably.
State $s_{i,j}$, $s_{i,j}^\top$ and $s_{i,j}^\bot$ have observations $\#, \top, \bot$, respectively. We note that a path  reaches $s_{i,j}^\top$ if $\alpha(x_i) = \top$ and (analogously for $\bot$).
From there onwards,  if the literal $x_i$ (or $\neg x_i$) occurs in clause $c_j$, we transition from $s_{i,j}^\top$ (or $s_{i,j}^\bot$, resp.) to $s_{i+1,j}'$, otherwise, we transition to $s_{i+1,j}$. A path ends in $t$ only via a state $s'_{i,j}$, which is only possible if the path visits a state corresponding to a literal in the clause. Thus, (conditionally) reaching state $s'_{i,j}$ with positive conditional probability means that the partial assignment $\alpha(x_1),\dots,\alpha(x_i)$ satisfies clause $c_j$, while reaching $s_{i,j}$ with positive conditional probability means that the clause is either unresolved or unsatisfied given the partial assignment.

Formally, we construct the HMM $\mc = (\Sts,\init, \ptrans, \Obs, \obsf, r)$ and set $\lambda = 1$:
\begin{itemize}
    \item
          $\Sts = \{ s_{i,j}, s_{i,j}^\bot, s_{i,j}^\top,
              s_{i,j}', s_{i,j}'^\bot, s_{i,j}'^\top
              \mid i \in \{1, \dots, n+1 \},  j \in \{ 1, \dots, m \} \} \cup \{ s_\init \}$.
    \item $\ptrans$ is given such that for all $i \in \{1,\dots, n\}, j \in \{1, \dots, m\}$:
          \begin{itemize}
              \item  $\ptrans(s_\init,s_{1,j}) = \frac{1}{m}$,
              \item  $\ptrans(s_{i,j},s_{i,j}^\top) = \ptrans(s_{i,j},s_{i,j}^\bot) = \frac{1}{2}= \ptrans(s'_{i,j},s_{i,j}'^\top) = \ptrans(s_{i,j}',s_{i,j}'^
                        \bot)$,
              \item $\ptrans(s_{i,j}^\top,s'_{i+1,j}) = \indicator{x_i = \ell_j^k \text{ for some k}}$, $\ptrans(s_{i,j}^\top,s_{i+1,j}) = \indicator{x_i \neq \ell_j^k \text{ for all k}}$,
              \item $\ptrans(s_{i,j}^\bot,s'_{i+1,j}) = \indicator{\neg x_i = \ell_j^k \text{ for some k}}$, $\ptrans(s_{i,j}^\bot,s_{i+1,j}) = \indicator{\neg x_i \neq \ell_j^k \text{ for all k}}$
              \item $\ptrans(s'_{n+1,j}, t) = 1 = \ptrans(s_{n+1,j}, f)$
          \end{itemize}
    \item $\Obs = \{\top, \bot, \#\}$ and $\obsf(s_{i,j}^\top) = (s_{i,j}'^\top) = \top$,
          $\obsf(s_{i,j}^\bot) = (s_{i,j}'^\bot) = \bot$,
          $\obsf(s_{i,j}) = \obsf(s'_{i,j}) = \# = \obsf(s_\init) = \obsf(t) = \obsf(f)$.
    \item $r(t) = 1$ and $r(s) = 0$ for all $s \in \Sts \setminus \{ t\}$.
\end{itemize}
The construction runs in polynomial time. The formal proof of its correctness follows the explanation above precisely.

\subsubsection{Approximability}

Approximating the verification of monitors is also hard, in the sense that it is hard to approximate the maximal risk that a monitor admits.

The maximal risk problem is an optimization problem
\begin{definition}[CTP Optimization Problem]
 Given an HMM $\mc$ with states in $S$, a unary encoded horizon $h$, a set of alarm states $A\subseteq\Sts$: \[\max_{\tau \in \lang{\mc}}~ \sum_{\pi\in\Pi^\mc_{=h}\mid\pi_\downarrow \in A}\Prob^\mc(\pi \mid \tau) \cdot r(\pi_\downarrow)\]\sj{check notation for paths of length $h$}
\end{definition}\sj{Sync with CTP definition}
\begin{lemma}
    \label{lem:ctpapxhard}
    The CTP Optimization Problem is APX-hard.
\end{lemma}
This follows from a strict reduction from MAX-3SAT, which is an inapproximable and APX-hard problem~\cite{DBLP:journals/jacm/Hastad01}. The construction coincides with the reduction in \Cref{lem:ctpdecnphard} by observing that the conditional probability to reach a $t$ state is given by $\nicefrac{1}{m}$ times the number of satisfied clauses, i.e., we can compute the maximal number of satisfied clauses in $\varphi$ on the HMM $\mc_\varphi$.
