\begin{abstract}
    Runtime monitors for hidden Markov models obtain a sequence of observations and must classify whether the system is safe or unsafe. That is,  the probability of being in a dangerous state conditioned on these observations exceeds some acceptable threshold.
    The key contribution of this paper is the first correct-by-construction synthesis method for such monitors, represented as finite state machines.
    First, we establish the coNP-hardness of checking whether a finite state machine  is indeed a correct monitor, i.e., a monitor without misclassifications.
    Second, we provide a reduction that reformulates the search for misclassifications into a standard probabilistic system synthesis problem.
    Third, we integrate the verification routine into an active automata learning routine to synthesise correct monitors.
    Fourth, we provide a prototypical implementation that shows the feasibility and limitations of the approach on a series of benchmarks.

\section{Introduction}
Runtime assurance is an essential ingredient in the deployment of safe autonomous systems.
Runtime monitors provide assurance by flagging potentially dangerous system behavior by observing a system execution.
More precisely, a monitor receives a stream of observations about the system and outputs a verdict, e.g., it raises an alarm that the system has left some safety envelope. A monitor is correct, if it correctly raises such alarms based on a formal specification.
Various challenges in creating correct runtime monitors for (semi-)autonomous systems have been identified~\cite{}, such as:
(1)~the state of the system is only partially observable, i.e., the stream of observations comes from sensor readings and does uniquely identify the state of a system,
(2)~the behavior of the system and/or the sensors may be subject to probabilistic uncertainty,
(3)~the monitor itself is subject to resource constraints (time, memory, etc), and
(4)~the monitor is itself safety-critical and should therefore be subject to extensive validation.
Challenges (1,2) can be (partially) addressed  by modelling the system as a hidden Markov model (HMM), Challenges (3,4) can be (partially) addressed by representing a monitor as, e.g., a (small) finite state machine.
Concretely, this paper focuses on the following question:
\emph{Is a given finite state machine a correct monitor for a given HMM?}
The paper studies the complexity of this problem, provides a practical verification procedure, and embeds this verification procedure into a learning framework.

\paragraph{What are HMMs?}
In this paper, we assume that the system including its sensors is adequately described as a discrete Hidden Markov Model (HMM).
Markov chains (MCs) describe system behavior subject to probabilistic uncertainty.
Paths through an MC are sequences of states that describe system executions.
% MCs can be seen as specifying a distribution over these paths. 
HMMs extend MCs by labelling their states with colours (aka \emph{observations}).
Intuitively, the observations can be used to model the information that the monitor receives.
In HMMs, every path can be lifted to a sequence of observations, which we call a \emph{trace}.
The trace associated to a system execution describes the information received by the monitor.


% where every state is labelled with a distribution over observations, see Fig.~\ref{}. Paths are sequences of states and MCs can be seen as specifying a distribution over these paths. An HMM extends MCs by assigning to every state a colour (an \emph{observation}). A trace is a sequence of these observations. 

\paragraph{Risk of a trace.}
In runtime monitoring, an essential task is to estimate whether the current system is dangerous. In the simplest setting, a specification describes a set of states is dangerous. The risk of a trace is then the probability of ending in a dangerous state, conditioned on the fact that the system execution matches the trace.
For a given trace, we may compute this risk, e.g., either via a reduction to model checking~\cite{} or by a (forward) filtering that tracks a distribution over the current states~\cite{}.


\paragraph{What are correct monitors?}
Often, we are not interested in this precise risk, but rather want to  partition the set of horizon-bounded traces into safe and unsafe traces:
Unsafe traces are traces whose risk does exceed some fixed \emph{acceptable threshold}, all other traces are safe.
Indeed, a \emph{correct} monitor obtains a trace and raises an alarm iff a trace is not safe. The key verification problem in this paper asks whether a monitor, represented as a deterministic finite automaton, accepts (i.e., raises an alarm) exactly the unsafe traces. We note that in this paper, we only consider traces that are bounded by some fixed horizon.

\paragraph{Novel complexity results.}
Deciding whether a monitor is correct is coNP-complete (\Cref{lem:complexity}).
In particular, we study the dual problem that asks whether a monitor misclassifies at least one trace.
We consider a simpler variant of this problem that asks whether there is a bounded trace in the HMM that is unsafe. \footnote{The results also apply to checking for safe traces.}
The problem is clearly in NP (assuming that the bound is unary encoded).
We show that this simplified problem is also strongly NP-hard (\Cref{lem:ctpdecnphard}) and the corresponding optimization problem, that asks to approximate the risk of the trace with the heighest risk, is APX-hard (\Cref{lem:ctpapxhard}).
Intuitively, APX-hardness means that the existence of an efficient approximation algorithm would have major complexity-theoretic implications.


\paragraph{Our verification approach.}
While deciding whether a monitor is correct is thus in general not tractable, we suggest to utilise recent advances in the synthesis of probabilistic systems.
We concentrate on proving the absence of missed alarms, i.e., we concentrate on showing that the monitor correctly identifies every unsafe trace. A similar reduction works for showing the absence of spurious alarm.
First, for a single trace, computing the risk can be reduced to computing a reachability probability in a MC that is intuitively a kind of product between a DFA that accepts exactly the trace and the HMM~\cite{}.
Thus, it is natural to reduce our problem to the question: Is there a DFA (accepting exactly one trace, accepted by our monitor)
such that the probability in the product MC exceeds a threshold.
If the answer is no, then the monitor has no missed alarms.
This decision problem can be solved using (exact) probabilistic model synthesis methods~\cite{} as implemented in the tool Paynt~\cite{}\footnote{Alternatively, in Section~\ref{sec:useparametersynthesis}, we show that we can construct a product of an acyclic DFA accepting sets of traces and the HMM. However, this yields partially observable MDP, where we must search for memoryless policies, similar to a construction in~\cite{DBLP:conf/tacas/BadingsVJSJ24}. This problem is equivalent to the synthesis problem and can therefore also be solved using Paynt~\cite{DBLP:conf/uai/Andriushchenko022}.}. We discuss our verification approach in \Cref{sec:monver}.

\paragraph{Our learning approach. }
Ultimately, we do not only want to verify the correctness of a monitor, but we want to construct such a monitor.
Above, we already mentioned that the monitors are formal languages. When considering bounded trace, these monitors are regular, and can thus be captured using DFAs.
Inspired by a recent short paper~\cite{}, we investigate learning these monitors:
We choose to do this using active automata learning~\cite{DBLP:journals/iandc/Angluin87,DBLP:journals/cacm/Vaandrager17}, which closely relates to other popular oracle-guided inductive synthesis loops~\cite{}.
To use active automata learning (AAL),
we must provide a \emph{membership oracle} that decides for a single trace whether the risk of the trace exceeds a threshold and an \emph{equivalence oracle} that decides whether a given hypothesis monitor is indeed correct. The membership oracle can be implemented by computing the risk for a single trace as above and comparing it to a threshold, while the equivalence oracle can be implemented by checking for the absence of missed and false alarms.
Practically, some minor modifications are necessary to embed our approach in  AAL framework: We have to handle the finite horizon, resolve some traces that are not relevant for the monitor, and we combine conformance and equivalence queries to boost performance.

\paragraph{Contributions.}
In summary, this paper provides a first framework to learn runtime monitors --- encoded as finite automata --- that are verified to be correct with respect to a given HMM. The main contributions are as follows:
(1)~We study monitor verification and show complexity-theoretic hardness of the underlying problem, even in an approximative setting.
(2)~We reduce the monitor verification to probabilistic model synthesis problems that exhaustively search for a counterexample.
(3)~We learn monitors by using the verifier above to answer equivalence queries in conjunction with active automata learning.
(4)~We demonstrate the feasibility and limitations of both the verifier and the learner on a series of benchmarks. Our prototype can handle HMMs with thousands of states, up to a hundred observations, and monitors with over 100 states.

\paragraph{Notation.}
Let $X$ be a finite set. A distribution $\mu$ over $X$ is a mapping $\mu\colon X \rightarrow [0,1]$ such that $\sum_{x \in X} \mu(x) = 1$. The set of all distributions over $X$ is written $\Distr(X)$.


\subsection{Models}
In this section, we briefly introduce MDPs, which are important later, and HMMs which are crucial to the problem statement. Details can be found in \cite{baierPrinciplesModelChecking2008}.


\begin{definition}[MDP]
    A \emph{Markov decision process} (MDP) is a tuple $\mdp = \left(\Sts, \init, \Act, \ptrans \right)$ with a  countable nonempty set  $\Sts$ of \emph{states},  the \emph{initial state} $\init \in \Sts$, the \emph{partial transition function} $\ptrans\colon \Sts \times \Act \rightharpoonup \Distr(\Sts)$.
\end{definition}
We use $\Act(s) = \{ \act \mid \ptrans(s, \act) \neq \bot \}$ as the set of enabled actions. We assume that there are no deadlocks, i.e., for every $s \in \Sts$, $\Act(s) \neq \emptyset$.
A path $\pi$ in an MDP is a sequence $s_0\act_0s_1\hdots \in (\Sts \times \Act)^{*} \times \Sts$, such that $a_i \in \Act(s_i)$ and $P(s_i, a_i)(s_{i+1}) > 0$ for every $i \geq 0$. %The set of all infinite paths is $\Pi^\mc$. 
The set of (finite) paths is denoted by $\Pi_{fin}^\mdp$ and the set of paths of at most length $h$ are $\Pi_h^\mdp$. If the MDP is clear from the context, the $\mdp$ is dropped.

A policy for an MDP is a function $\sigma\colon \Pi^\mc \to \Act$, which given a path through the MDP, gives a action. Given a policy, a MDP can be induced into a MC. Policy synthesis for an MDP for a property $\phi$ thus entails finding a policy for an MDP such that the induced MC entails $\phi$.

A \emph{Markov chain} (MC) is an MDP where $|\Act(s)| = 1$ for every state $s \in \Sts$. We simplify notation and write MCs also as tuple $\left(\Sts, \init,  \ptrans \right)$ and write $\ptrans(s)$ to refer to the unique distribution $\ptrans(s, \act)$.  Paths in a MC can be simplified to be sequences of (only) states.

\begin{definition}
    A deterministic finite automaton (DFA) is a tuple $\dfa=\left(\StsD, \alphabet, \trans, \init, \final\right)$. $\StsD$ is a finite set of states, $\alphabet$ is an alphabet, $\trans\colon \StsD \times \alphabet \rightharpoonup \StsD$ is a (partial) transition function, $\init$ is the initial state, and $\final \subseteq \StsD$ is a set of accept (or: final) states.
\end{definition}
Let $\varepsilon$ denote the empty word. We lift the transition relation to words, which is defined recursively: $\trans^*(q, \varepsilon) = \varepsilon$ if $a \in \alphabet$ and $\trans^*(q, w \cdot a) = \trans(\trans^*(q,w), a)$.

The \emph{language} $\lang{\dfa}$ of a DFA $\dfa$ is the set of all words accepted by the $\dfa$, $\lang{\dfa} = \{ w \in \alphabet^* \mid \trans^*(\init, w) \in \final \}$.

\subsection{Formal Problem Statement}
The monitor verification problem consists of two sub problems. The no false alarms problem, and the no missed alarms problem. Both of these sub problems entail finding the trace with either the minimum or maximum risk. We will discuss the missed alarm problem in the following sections. In \cref{ssec:nfa} we will adapt to the no false alarms problem.

The no missed alarms problem entails ensuring that all traces from the MC, which are not in the monitor, have at most a risk of $\lambda$.

\newcommand{\safetraces}[2]{\mathbb{S}_{#1}^{#2}}
\newcommand{\unsafetraces}[2]{\mathbb{U}_{#1}^{#2}}
\newcommand{\missedalarms}[2]{\mathsf{mA}_{#1}^{#2}}
\newcommand{\falsealarms}[2]{\mathsf{fA}_{#1}^{#2}}
\newcommand{\defeq}{:=}


\begin{definition}[Risk of a trace]
    Given a HMM $\mc$. The risk of trace $\tau \in \lang{\mc}$ is: \[ R_r(\tau) \defeq \sum_{\pi\in\Pi^h_\mc}\Prob(\pi \ |\ \tau) \cdot r(\pi_{\downarrow}).\]
\end{definition}
The risk of a trace can be used to partition the set of all traces.
\begin{definition}[Safe traces/Unsafe traces]
    Given a HMM $\mc$, a \emph{safe threshold} $\lambda_s \in \mathbb{R}$ and an \emph{unsafe threshold} $\lambda_u \in \mathbb{R}$ with $\lambda_s \leq \lambda_u$.
    A trace $\tau \in \lang{\mc}$ with $R(\tau) > \lambda_u$ is an \emph{unsafe trace} (for $\mc$ and $\lambda_u$). Conversely, trace $\tau$ is a \emph{safe trace}, if $R(\tau) < \lambda_s$ (for $\mc$ and $\lambda_u$).
\end{definition}
In the definition above, we deliberately do not require $\lambda_s = \lambda_u$. In particular, by picking $\lambda_s < \lambda_u$, some traces are neither safe nor unsafe. We write $\safetraces{\mc,\lambda_s}{\leq h}$ to denote the set of safe traces of length at most $h$ and $\unsafetraces{\mc,\lambda_u}{\leq h}$ for the set of unsafe traces of length at most $h$. We often omit the sub/superscripts.

We are now ready to define correct monitors.
\begin{definition}[Monitor]
    A DFA $\dfa$ is a monitor for HMM $\mc$ if the alphabet for $\dfa$ coincides with the observations in $\mc$.
\end{definition}
\begin{definition}[Missed/False alarms]
    Given a monitor $\dfa$ for HMM $\mc$, a horizon $h$, and thresholds $\lambda_s \leq \lambda_u \in \mathbb{R}$, the set of \emph{missed alarms} is $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa) \defeq \unsafetraces{\mc,\lambda_u}{\leq h} \setminus \lang{\dfa}$. The set of \emph{false alarms} is $\falsealarms{\mc,\lambda_s}{\leq h}\!(\dfa) \defeq \safetraces{\mc,\lambda_s}{\leq h} \cap \lang{\dfa}$.
    % a safe trace $\tau \in \Obs^{\leq h}$ with $R(\tau) > \lambda_s$ and $\tau \not\in \lang{\dfa}$ is a \emph{missed alarm}. Conversely, trace $\tau$ is a \emph{false alarm}, if $R(\tau) < \lambda_u$ and $\tau \in \lang{\dfa}$. 
\end{definition}
We drop the sub/superscripts whenever possible. In terms of a classification problem, missed alarms can be seen as false negatives while false alarms are false positives.
\begin{definition}[Correct monitor]
    Given thresholds $\lambda_s,\lambda_u$ and horizon $h$, a monitor $\dfa$ for $\mc$ is \emph{correct} if $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa) = \emptyset = \falsealarms{\mc,\lambda_u}{\leq h}\!(\dfa)$.
\end{definition}
Equivalently, a monitor is correct if it raises an alarm for every unsafe trace and for no safe trace:
\begin{corollary}[Correct monitor]
    A monitor $\dfa$ for HMM $\mc$ with $ \unsafetraces{\mc,\lambda_u}{\leq h} \subseteq \lang{\dfa} \subseteq \alphabet^* \setminus \safetraces{\mc, \lambda_s}{\leq h}$ is a correct monitor.
\end{corollary}

\begin{mdframed}\textbf{Problem statements.} This paper tackles the following problem statements:
    Given HMM $\mc$, thresholds $\lambda_s,\lambda_u$ and horizon $h$,
    \begin{enumerate}
        \item Given monitor $\dfa$ for $\mc$, are there missed alarms, i.e., is $\missedalarms{\mc,\lambda_s}{\leq h}\!(\dfa)\! = \emptyset$?
        \item Given monitor $\dfa$ for $\mc$, are there false alarms, i.e., is $\falsealarms{\mc,\lambda_u}{\leq h}\!(\dfa)\! = \emptyset$?
        \item Find a correct monitor $\dfa$ for $\mc$ w.r.t.\ $\lambda_s, \lambda_u$ and $h$.
    \end{enumerate}
\end{mdframed}
Problem statements 1 and 2 together allow checking whether a monitor is correct.
\begin{remark}
    A correct monitor must exist, as $\unsafetraces{}{\leq h}$ is finite and thus regular.
\end{remark}

\section{Monitor Verification}
\label{sec:monver}
In this section we will discuss our solution to the monitor correctness problem. We focus on the no missed alarms problem with traces of exactly the length of the horizon. We will solve this using policy synthesis of colored MDPs. At the end, we will generalize our construction to both the no false alarms problem and traces of length at most the horizon.

\subsection{Colored MDPs}
Policy synthesis for colored MDPs has seen recent successes\cite{andriushchenkoPAYNTToolInductive2021}. We will reduce our problem into a colored MDP policy synthesis problem.

\begin{definition}[Colored MDP]
    Given an MDP $\mdp$, with states in $\Sts$, a \emph{colored MDP} is a tuple $\mdpc = (\mdp, C, c\colon \Sts \to C)$. $C$ is the set of colors, and $c$ maps states to colors.
\end{definition}
We will only consider policies for colored MDPs, which act equally for states of the same colors \footnote{Colored MDPs with color consistent policies coincide with memoryless policies for POMDPs. However, since POMDPs often consider not only memoryless policies, we will use colored MDPs.}.
\begin{definition}[Color consistent scheduler]
    A scheduler $\sigma$ for a colored MDP $\mdpc$ with states in $\Sts$ is color consistent if for every two states $s$ and $s'$
    \[ c(s) = c(s') \Rightarrow \sigma(s) = \sigma(s') \]
\end{definition}
We now present the theorem we focus on in this section.
\begin{theorem}
    Given a HMM $\mc$ with a monitor $\dfa$, threshold $\lambda_s$, horizon $h$, and risk $r$, there exists a transformation $f$ into colored MDPs, such that iff there exists a color-consistent policy for $f(\mc, \dfa, \lambda_s, h, r)$ with $\Prob>\lambda$ then there is a trace $\tau \in \missedalarms{}{}(\dfa)$
\end{theorem}
Following from the transformation we will present we also have:
\begin{corollary}
    There is a one to one correspondence between a found $\tau$ and a color consistent policy.
\end{corollary}
Remark that the no missed alarms problem contains several elements which are no longer present in the colored MDP. We will have to incorporate the monitor, horizon, observations, and risk. Furthermore, recall that the no missed alarms problem is stated in terms of a conditional probability on the trace. Policy synthesis cannot take conditional probabilities into account, thus we will also incorporate this in the transformation.

For clarity’s sake, we describe our transformation in several steps. Only at the end do we move from HMMs to colored MDPs. Here we give a quick outline of the algorithm. In \cref{ssec:ctp} we will first move the monitor into the HMM. Secondly we will transform the HMM to both include the horizon and the risk. In \cref{ssec:polsyn} we resolve the conditioning on traces, while also removing the observations from the model, by transforming into colored MDPs.

First, we show how the dual of the no missed alarms problem, the exists missed alarm problem, can be transformed into the simpler \emph{Conditional trace probability} (CTP) problem. Finally, we will further simply the problem by incorporating both the horizon and risk such that we are left with acyclic HMMs. We will discuss this intermediate problem as the Acyclic CTP (ACTP) problem.

The exists missed alarm problem, questions whether there is a trace $\tau$ such that $\tau \in \missedalarms{}{}(\dfa)$. Obviously, when such a trace exists, $\missedalarms{}{}(\dfa) \neq \emptyset$, thus disproving the no missed alarms problem for $\dfa$.

\subsubsection{CTP problem}
Before we combine the HMM and the monitor, we have to modify the monitor. Since we are looking for missed alarms, we are interested in all traces that are not in the monitor. We thus take the complement of the monitor.

We transform the HMM using the Product Markov Chain construction from \citeauthor{baierPrinciplesModelChecking2008} \cite{baierPrinciplesModelChecking2008}. It combines a DFA and a MC by taking the product of the states and using the obvious initial states and transition function. We extend this to HMMs, by letting the observation function observe the model states. This extension allows us to apply the Product Markov Chain construction to our model HMM and complemented monitor DFA, resulting in a product HMM. We also keep track of all resulting states for which the complement monitor accepts in a set of alarms states $A$ of the product HMM.

The resulting product HMM can be seen as the model of a simplified version of the no missed alarms problem, the CTP problem.

\begin{definition}[CTP Decision Problem]
    Given an HMM $\mc$ with states in $S$, a unary encoded horizon $h$, a set of alarm states $A\subseteq\Sts$, a risk function $r\colon S \to \RR^+$, and a threshold $\lambda_s \in (0,1]$, find a trace $\tau \in \lang{\mc}$ such that $$\sum_{\pi\in\Pi^\mc_H\mid\pi_\downarrow \in A}\Prob^\mc(\pi \mid \tau) \cdot r(\pi_\downarrow) \geq \lambda$$
\end{definition}
As we will be shown in \cref{sec:compl} the CTP problem is NP-complete. By using the above reduction to exists missed alarm problem, it is easy to show that the no missed alarms problem is co-NP-complete.

We will build up the ACTP HMM by creating its transition function $\ptrans'$, all other elements of the ACTP HMM are easily derivable.
To include the horizon into the ACTP HMM, we unroll the CTP HMM over the horizon $h$. The ACTP HMM has states in $\{1,\ldots,h\}\times\Sts$. The ACTP transition function $\ptrans'$ on non-horizon states behaves as expected:
\[\ptrans'(\langle i, s\rangle, \langle i+1, q\rangle) = \ptrans(s, q)\]
with $i \in \{1,\ldots,h-1\}$. The horizon states, $\langle h, s \rangle$ for $s \in \Sts$, transition to one of three states, $\alarms$, $\safes$ or $\igs$ state. Horizon states in $A$ will transition to $\alarms$ and $\safes$ according to the \todo{Mention normalizing risk more prominently somewhere?} normalized risk. Horizon states not in $A$ transition to $\igs$ instead.
\begin{align*}
    \ptrans'(\langle h, s\rangle, \alarms) & = \begin{cases}
                                                   r'(s) & \text{if } s \in G \\
                                                   0     & \text{otherwise}
                                               \end{cases}     \\
    \ptrans'(\langle h, s\rangle, \safes)  & = \begin{cases}
                                                   1 - r'(s) & \text{if } s \in G \\
                                                   0         & \text{otherwise}
                                               \end{cases} \\
    \ptrans'(\langle h, s\rangle, \igs)    & = \begin{cases}
                                                   1 & \text{if } s \notin G \\
                                                   0 & \text{otherwise}
                                               \end{cases}
\end{align*}
The $\alarms$, $\safes$ and $\igs$ states become sink states. The $\alarms$ and $\safes$ sates get the $end$ observation, while the $\igs$ state gets the $\ig$ observation.

We add an $\igs$ state, instead of redirecting all traces we don't care about to the $\safes$ state, to more easily modify the transformation for the no false alarms problem.

\begin{remark}
    Note that we ignore all states in $A$ that are not horizon states. However, since we only consider traces which are of length exactly the horizon, we only need to consider the alarm states which are encountered after horizon steps.
\end{remark}

Using the above transformed HMM we can define the ACTP problem.
\begin{definition}[Asyclic CTP problem]
    Given an acyclic HMM $\mc$ with observations $Z$, an alarm state $a$, and a threshold $\lambda \in (0,1]$, is there a trace $\tau \in Z^\star$ such that $\sum_{\pi\in\Pi^\mc}\Prob^\mc(\pi a \mid \tau) \geq \lambda$.
\end{definition}
In the ACTP HMM we no longer consider the risk of a trace, instead we only reason about the probability of reaching the alarm state. The ACTP problem thus contains a threshold between in the interval $(0,1]$. We also no longer need to consider the horizon, as all traces consist of a horizon number of observation in $Z$ and an infinite sequence of either $end$ or $\ig$ observations.

In the ACTP HMM we only have the observations and conditional probability on a trace left. To incorporate this into the model we will use one novel idea. Instead of transitioning to a state and observing it, we will allow the model to choose what observation to observe. We thus move away from HMMs and into MDPs. This has two results. We will first discuss how to create this MDP. Afterward, we show how to ensure how, traces and policies can be made equivalent by coloring the MDP.

We construct the policy synthesis MDP where the transition function $\ptrans''$ is defined as
\begin{align*}

    \begin{cases}
        \ptrans'(s, q)                                     & \text{if } \obsf(q) = z \\
        \sum_{q'\in S\ |\ \obsf(q') \neq z} \ptrans'(s,q') & \text{if } q = \iota    \\
        0                                                  & \text{otherwise}
    \end{cases}
\end{align*}
Thus we transition normally to a state if the action and observation of the target state are equal, otherwise we set the transition probability to zero. All the remaining probability mass is redirected towards the initial state. \todo{Should we give a reason why this is correct? The above construction allows for conditioning on a trace $\tau$ by constructing a policy $\sigma$ that selects the $i$th observation of $\tau$ in the state with step $i$.
\begin{definition}[Trace consistent policy]
    Given two sets $\Sts$ and $Z$ and an MDP $\mdp$ with states in $\NN\times\Sts$, and actions in $Z\cup\{end\}$. Together with a trace $\tau \in Z^\star$. A \emph{trace consistent policy} is of the form
    \begin{align*}
        \sigma_\tau(\langle i, s \rangle) = \begin{cases}
                                                \tau^{(i)} & \text{if } i \leq |\tau| \\
                                                end        & \text{otherwise}
                                            \end{cases}
    \end{align*}
\end{definition}
\begin{corollary}
    There exists a map $t(\sigma_\tau) = \tau$ which, given a trace consistent policy $\sigma_\tau$ finds its associated trace $\tau$.
\end{corollary}
However, when finding the maximal trace, we might not find a trace consistent policy. To fix this we color the MDP such that color-consistent policies coincide with trace consistent policies on our MDPs. As colors $C$, we take the set $\{1,\ldots,h\}$ of steps until the horizon, and as coloring function take the projection $c(\langle i,s \rangle) = i$. Given that all $end$ states sinks, color consistent policies with the trace consistent policies.

\end{remark}

By construction, there is a trace with risk $1$ iff there is a satisfying assignment to $\varphi$. Note that there is a trace with risk $1$ iff there is a trace where all corresponding paths end in state $t$.

Before we give a formal definition of $\mc_\varphi$, we give some intuition, supported by Fig.~\ref{fig:hmmgadget}.
We represent assignments $\alpha\colon X \rightarrow \{0,1\}$ by traces through the HMM $\#\# \cdot \alpha(x_0) \cdot \# \cdot \alpha(x_1) \dots \cdot \#$.
This trace matches with exactly $m$ paths through the HMM, one for each clause. That is, intuitively, $\mc_\varphi$ randomly selects a clause $c_j$ with probability $\nicefrac{1}{m}$ in the initial state and then `evaluates' $c_j$ with respect to an assignment $\alpha$, by starting in $s_{1,j}$.

That is, a path through clause $c_j$ `reads' variable $x_i$ in state $s_{i,j}$ and transitions to $s_{i,j}^\top$ and $s_{i,j}^\bot$, both with probability $\nicefrac{1}{2}$.
State $s_{i,j}$, $s_{i,j}^\top$ and $s_{i,j}^\bot$ have observations $\#, \top, \bot$, respectively.
Thus, the conditional probability to reach $s_{i,j}^\top$ is $\nicefrac{1}{m}$ if $\alpha(x_i) = \top$ and $0$ otherwise.
Then, if the literal $x_i$ (or $\neg x_i$) occurs in clause $c_j$, we transition from $s_{i,j}^\top$ (or $s_{i,j}^\bot$, resp.) to $s_{i+1,j}'$, otherwise, we transition to $s_{i+1,j}$.
The idea is that (conditionally) reaching state $s'_{i,j}$ with positive cconditional probability means that the partial assignment $\alpha(x_1),\dots,\alpha(x_i)$ satisfies clause $c_j$, while reaching $s_{i,j}$ with positive conditional probability means that the clause is either unresolved or unsatisfied given the partial assignment.
